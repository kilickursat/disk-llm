Research Notes

Why publish this anyway.

Disk-LLM is a research repo. The point is not to only show flattering artifacts. The point is to make the storage path, runtime path, and benchmark truth visible enough that improvements can be measured honestly.

Evidence over optics

The most useful finding from this rerun is not that Qwen is fast today. It is that the repo now has a trustworthy full-model baseline to improve against. The next work items are already clear: optimize the HF CPU image path, apply only small Disk-LLM runtime tweaks, rerun postfix as v4, and then republish the updated snapshot.

  • HF CPU image optimization comes first because the current Modal baseline still pulls an oversized Torch stack for a CPU benchmark.
  • Disk-LLM changes stay small and local so the project's native NumPy memmap identity remains intact.
  • The next public benchmark refresh should be based on a new validated postfix v4 run, followed by a matching prefetch comparison.
